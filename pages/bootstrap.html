Bootstrap is a very useful tool for developing responsive web pages, which allows sites to look good and work correctly on different devices and screens. In addition, it offers a wide variety of tools and components so that developers can create attractive and functional websites with greater speed, ease and efficiency.
Interactivity
Bootstrap provides great interactivity on the page through a series of components that facilitate interaction with the user, such as navigation menus, progress bars, and page controls.
Responsive design
The main goal of Bootstrap is to build responsive websites for mobile devices. In a very simple and organized way, it allows you to design pages specially adapted to work on smartphones, tablets and desktops.
Component Library
Bootstrap offers a large number of components to improve user communication, such as different types of alerts, a slide carousel for easy image viewing, and a navigation bar that allows for multiple configurations.
Advantages
  • Easy to use and saves time - It allows you to design a website using different combinations of HTML, CSS and JavaScript. It uses modern technologies like HTML5, CSS3 and jQuery, and integrates with JavaScript libraries. It comes with ready-made styles and templates. It comes with complete documentation that makes it easy to use.
  • Responsive - Its templates are easily customizable and responsive. It includes a useful grid system for layout by columns that adapt to different screen sizes and devices.
Disadvantages
  • Compatibility - Changing versions can be difficult if deep modifications have been made. It is recommended to work with Bootstrap from the beginning of the project to avoid compatibility issues. It is not a lightweight framework and some functionality requires the use of JavaScript and jQuery.
  • Layout constraints - The layout needs to be adapted to a 12-column grid system, which may require specific changes to Bootstrap's default styles.
  • Can be heavy - The files generated by Bootstrap and end results can be quite heavy and result in slower loading times, as well as battery drain issues.
  • Unoriginal design - Without a lot of modification, everything built with Bootstrap will have a very similar look.
Bootstrap can be easily installed by visiting the official page and downloading the package files. The installation files include CSS style sheets, JavaScript files, and fonts, and can be adapted to any type of web project. To work with Bootstrap, simply add the style sheets and the JavaScript file in the header or before the closing tag of the HTML file. Alternatively, it can be used using Bootstrap CDN links that can be included in the project for free.
Sites built with Bootstrap will automatically adjust to the size of the screen they are viewed on. This is possible thanks to the inclusion of the Bootstrap Grid system.
The grid system divides the screen into twelve virtual columns and allows developers to specify how elements should be displayed on each screen size. For example, on a small screen, certain elements can be specified to stack vertically to make them easier to read and navigate on mobile devices.
To implement a responsive layout in Bootstrap, you can use the column size classes, such as col-sm for small screens, col-md for medium screens, and col-lg for large screens. These classes can be applied to any HTML element, such as a div or an image, to define how it will display on different screen sizes.
For example, an element can be specified to have a width of 12 columns (full width of screen) on mobile devices, but only 3 or 4 columns on computer screens.
Bootstrap offers a large number of prebuilt components that web developers can use in their projects. These components include navigation menus, carousels, pricing tables, and many others. On its official page, Bootstrap offers examples of these components, which makes it easy to implement. Some of the most used components are:
  • carousel
  • navbar and navs
  • jumbotron
  • breadcrumbs
  • pagination
  • tooltips
  • forms
  • cards
For instance, a navbar and nav components allow you to create a responsive navigation bar with different elements such as links and dropdowns. The navbar elements are created using HTML lists and styled with Bootstrap-specific classes. Bootstrap's inbuilt JavaScript can be used to activate the dropdown functionality in the navigation bar.
There are two major layouts for Bootstrap:
Fixed Layout
This uses the bootstrap .container class for the layout. The fixed-layout has specific pixel width values that change with the help of media queries, which makes the site responsive. Fixed layout resizes at intervals following the specified pixel values.
Fluid Layout
This uses the bootstrap .container-fluid class for the layout. This layout uses proportional values such as percentages for blocks of content, images, or any other item. Fluid layout continuously resizes as you change the width of your browser by any amount, leaving no extra empty space on the sides. Hence it is named “fluid layout”.
Containers are the most basic layout element in Bootstrap and are required when using our default grid system. Bootstrap comes with three different containers.
Fixed layout containers:
  • .container, which sets a max-width at each responsive breakpoint
  • .container-{breakpoint}, which is width: 100% until the specified breakpoint. The breakpoint can be (sm/md/lg/xl). See here for details.
Fluid layout container:
  • .container-fluid, which is width: 100% at all breakpoints
Bootstrap includes predefined button styles, each with its own style and/or semantic purpose:
  • Provides more visual weight to a button while representing a primary action for visitors to take on the page. E.g. homepage.
  • Provides more visual weight to a button while representing a secondary action for visitors to take on the page. E.g. demo
  • Indicates a successful or positive action. E.g. at the completion of a form.
  • A potentially dangerous action. E.g. to ensure visitors review their information before submitting.
  • Represents a warning. For example, the warning button can indicate that some required form fields were missed
  • An informational alert. For example, on an account registration form, an info button might indicate that a user can click to edit their profile.
  • A great option for web pages with dark background colors
  • Works well for web pages with light background colors.
